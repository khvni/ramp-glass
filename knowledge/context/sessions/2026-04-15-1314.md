## Decisions made

- **Preference storage**: keep it in the existing layout row JSON instead of adding schema or a new table. Simpler, local-first, and compatible with current repo patterns.
- **Chat identity**: use stable `chat` / `chat-N` panel IDs so each pane gets its own OpenCode session state and survives layout save/restore.
- **Default behavior**: auto-open stays enabled by default, including for legacy layout rows that predate the toggle.
