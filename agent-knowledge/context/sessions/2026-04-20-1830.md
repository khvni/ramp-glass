# 2026-04-20 18:30 — SQL execute permission for chat/memory path

## What happened

Sending a message from the Chat pane failed with:

`sql.execute not allowed. Permissions associated with this command: sql:allow-execute`

Root cause: the main window capability only granted `sql:default`, and in Tauri v2 that default set is read-only (`load`, `select`, `close`). Chat hits SQLite writes through the memory/layout stores, so `plugin:sql|execute` was denied.

## What changed

- Added `sql:allow-execute` to `apps/desktop/src-tauri/capabilities/default.json`.

## Verification

- `pnpm --filter @tinker/desktop typecheck`
- `pnpm --filter @tinker/desktop test`

## Note

`sql:default` is not enough for any code path that calls `database.execute(...)`. Future write paths need `sql:allow-execute` explicitly.
