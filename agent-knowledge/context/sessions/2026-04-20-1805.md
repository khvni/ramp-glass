# 2026-04-20 18:05 — `dev:web` startup crash guard

## What happened

`pnpm dev:web` was crashing on the startup error card with `Cannot read properties of undefined (reading 'invoke')`.
Root cause: `apps/desktop/src/renderer/App.tsx` booted the main app through plain Vite, but immediately called Tauri APIs (`invoke`, notification plugin, SQLite-backed scheduler/memory effects) even when `window.__TAURI_INTERNALS__` was absent.

## What changed

- Added `apps/desktop/src/renderer/runtime.ts` with a small Tauri-runtime detector plus a browser-preview notice string.
- Updated `App.tsx` to:
  - short-circuit boot into a browser-preview ready state when Tauri internals are missing
  - skip native-only effects (vault watch, scheduler start, memory sweep) in browser preview
  - guard native actions with explicit errors instead of falling through to `undefined.invoke`
  - keep browser preview on First Run instead of mounting the SQLite-backed workspace shell
- Updated `panes/FirstRun.tsx` to show the browser-preview notice and disable native-only buttons when running under `pnpm dev:web`.
- Noted the behavior in `agent-knowledge/context/tasks.md` under FirstRun UX.

## Verification

- `pnpm --filter @tinker/desktop typecheck`
- `pnpm --filter @tinker/desktop test`
- Live check against `http://127.0.0.1:1420` via Playwright:
  - startup crash card no longer renders
  - First Run renders with the browser-preview notice
  - only console error was an unrelated `favicon.ico` 404

## Follow-up

- If we want a richer `dev:web` experience later, the next step is a browser-safe in-memory host/store shim so the workspace shell can mount without Tauri. This change intentionally stops at “clear preview, no crash”.
