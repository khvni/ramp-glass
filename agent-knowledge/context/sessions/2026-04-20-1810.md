# 2026-04-20 18:10 — `dev:desktop` port reuse fix

## What happened

`pnpm dev:desktop` failed whenever `pnpm dev:web` was already serving `http://127.0.0.1:1420`.
Root cause: `apps/desktop/src-tauri/tauri.conf.json` used a raw `beforeDevCommand`:

`pnpm exec node ./scripts/prepare-opencode-sidecar.mjs && pnpm exec vite --host 127.0.0.1 --port 1420`

That always tried to bind `1420`, so Tauri aborted before startup if a valid Tinker Vite server was already running.

## What changed

- Added `apps/desktop/scripts/dev-web-or-reuse.mjs`.
- `beforeDevCommand` now points at that wrapper instead of invoking Vite directly.
- The wrapper:
  - runs the existing sidecar prep step
  - checks whether `1420` already has a listening process
  - validates the existing server over HTTP when possible
  - reuses an existing Node/Vite-style listener on `127.0.0.1:1420` instead of failing
  - only starts `vite --host 127.0.0.1 --port 1420` when nothing valid is already listening

## Verification

- `pnpm exec node ./scripts/dev-web-or-reuse.mjs`
  - output: `Reusing existing node listener on http://127.0.0.1:1420; HTTP validation was unavailable (fetch failed).`
- `pnpm --filter @tinker/desktop typecheck`
- `pnpm --filter @tinker/desktop test`

## Note

The local shell sandbox blocked direct localhost fetches with `EPERM`, so the wrapper includes a process-level fallback (`lsof`) instead of relying purely on HTTP probing.
