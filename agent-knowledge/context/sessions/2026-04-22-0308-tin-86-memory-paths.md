# TIN-86 — per-user memory path sync

Contributor: khvni
Agent: Codex

## Scope
- Deliver `TIN-86` for [[28-mvp-identity]].
- Ensure signed-in users always get a memory subdir, and emit a path-change signal when the active user or memory root changes.

## Changes
- Added active-memory-path event helpers to `packages/memory/src/memory-paths.ts`:
  - `resolveActiveMemoryPathChange`
  - `emitActiveMemoryPathChanged`
  - `subscribeActiveMemoryPathChanged`
- Added coverage in `packages/memory/src/memory-paths.test.ts` for unchanged-path no-ops, user-switch details, and subscriber notification.
- Updated `apps/desktop/src/renderer/App.tsx` to:
  - seed the active-user memory ref during boot
  - keep creating per-user memory subdirs on sign-in via `getActiveMemoryPath(...)`
  - re-resolve the active memory path whenever `currentUserId` changes
  - re-resolve and emit again when `subscribeMemoryPathChanged` fires after a memory-root move

## Verification
- `pnpm -r typecheck`
- `pnpm -r lint`
- `pnpm -r test`

## Notes
- This keeps TIN-86 scoped: it adds the event primitive and App wiring, but does not pre-implement M6.9/M7.7 subscribers beyond the existing memory-root hook.
- The active-path emitter is separate from the existing root-change emitter so Settings can keep listening for root updates without mistaking `<memory_root>/<user-id>` for the root itself.
